MacOS でのメモリ管理

説明

デフォルトでは、Bink は MacOS のメモリ機能を直接呼び出して、メモリ管理を MacOS で処理します。ただし、Bink の割り当て関数や解放関数を BinkSetMemory 関数に置き換えることもできます。

MacOS から直接メモリを割り当てる場合、Bink は次の順番でメモリの割り当てを試みます。

1) 最初に、アプリケーションのメモリ領域からの割り当てを試みます。

2) 次に、MultiFinder の一時メモリ ブロックからの割り当てを試みます。

3) 最後に、システムの一時メモリ プールからの割り当てを試みます。

通常は、この一連の処理によってすべての領域が調べられ、必要なメモリが必ず見つかります。

BinkGetSummary 関数を使用すると、メモリの使用量や最大使用量を確認できます。
