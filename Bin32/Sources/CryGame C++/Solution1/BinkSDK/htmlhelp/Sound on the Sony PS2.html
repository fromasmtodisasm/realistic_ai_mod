Sound on the Sony PS2

Discussion

For sound support on the Sony PS2, Bink supplies an high-level convenient API that interfaces to the IOP chip (which in turn talks to the SPU sound output chips). This API is described in the "The RAD_IOP API for the Sony PS2" section.

The RAD_IOP section describes this in more detail, but basically, Bink simply needs one of the two PCM output channels that each SPU on the PS2 supports. You can specify which one to use in the BinkSoundUseRAD_IOP call.

Bink pre-mixes all of its audio together and then passes it down in one SPU channel. For example, you can play multiple Binks with multiple sound tracks and it will still just use a single hardware sound channel. This sound mixing step also does sound format conversion, so you can use whatever frequency and format of audio in your Bink files, and we will automatically convert it to the PS2's hardware format of 48,000 Hz, 16-bit, stereo output.

Other than this PCM sound channel, we don't touch any of the rest of the sound functionality. This means you can use any API (like Multistream) to control the other 48 ADPCM voices or even the other PCM channel. We expect that we would work with any sound output middleware, assuming they don't pound on the same hardware voice that you told us to use.

If you don't want to use the RAD_IOP API, then you can definitely write this audio support yourself, just drop us an email, and we can send you sample code that you can modify (we do recommend just using the RAD_IOP library, though - it is robust and is designed not to get in the way of anything else.
