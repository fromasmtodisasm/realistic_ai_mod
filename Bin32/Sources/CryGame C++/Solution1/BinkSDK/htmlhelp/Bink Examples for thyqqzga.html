Bink Examples for the Sony PS2

Discussion

The Bink SDK supplies several example programs for Sony PS2. To create these examples with SN Systems or CodeWarrior, just create a new project and include the appropriate C source file (ExamPbb.C, or ExamPTex.C), and then add the Bink for Sony PS2 static library to the project (BinkPS2.a).

The following is a list of the example files that you can experiment with:

For an example of using Bink to display a video directly to the Playstations's back buffer, see the "ExamPbb.C" file. This example is really simple. It uses the EE to decompress a frame and then convert it into a RGB32 memory buffer. Then it uses the DMA system to copy the RGB buffer into the PS2 backbuffer. When the transfer is completed, we requested that the page be flipped on the next vertical blank. All standard PS2 stuff.

For an example of using Bink to decompress onto a texture and then render the texture to the screen, see the "ExamPTex.C" file. This example uses a little more complicated code to upload each video frame onto a RGB32 texture that is then rendered to the screen. This is also a simple example - way more PS2 interfacing code than actual Bink code.
