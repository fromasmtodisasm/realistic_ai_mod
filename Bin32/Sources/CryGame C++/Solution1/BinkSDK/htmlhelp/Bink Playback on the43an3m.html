Bink Playback on the Nintendo GameCube

Discussion

On the Nintendo GameCube, you don't have to share the CPU with the OS at all, so the typical Nintendo GameCube Bink playback loop is very simple. Bink handles timing with the BinkWait function, so we just need to check this function while we're playing a video.

So, a simple example of Nintendo GameCube playback would look something like this: 


while ( 1 )
{
  // decompress a frame 
  BinkDoFrame( bink );


  // skip frames if we need to...
  while ( BinkShouldSkip( bink )
  {
    BinkNextFrame( bink );
    BinkDoFrame( Bink );
  }


  // draw the video textures here, but don't show the frame yet
  Draw_Bink_textures( ... );


  // check input, process other tasks - whatever
  while ( BinkWait( bink ) )
    do_idle();


  // flip the buffers and show the video frame
  SwapBuffers();


  // is the video done?
  if ( bink->FrameNum == bink->Frames )
    break;

               
  // move to the next frame
  BinkNextFrame( bink );  
}
